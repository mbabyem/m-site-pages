# m-site-pages

## 项目简介

`m-site-pages` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅用于存放和展示各类静态页面素材。  
适合作为个人作品集、临时页面存档或轻量级页面托管使用。

## 目录说明

```
m-site-pages/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面所需的静态资源（CSS、JS、图片等）
├── index.html      # 仓库首页导航（可选）
└── README.md       # 本文件
```

- `pages/` 目录下按页面名称或用途组织子文件夹，每个页面独立存放。
- `assets/` 目录用于存放多个页面共用的资源，避免重复引用。
- 仓库根目录可放置一个简单的 `index.html` 作为页面列表导航，也可留空。

## 页面归档说明

- 每个 HTML 页面应保持独立完整，可自包含运行（或仅依赖 `assets/` 中的资源）。
- 页面文件命名建议使用英文小写，以短横线连接，例如 `about-me.html`。
- 如需在 GitHub Pages 上直接访问页面，请确保仓库已启用 Pages 功能，并将 `index.html` 或 `pages/` 下的文件配置为根目录。
- 本仓库不限制页面内容类型，但请遵守所在平台的使用条款。

## 维护说明

- 欢迎通过 Pull Request 提交新的页面或改进现有页面。
- 提交前请确保页面无外部恶意链接，资源来源合法。
- 仓库维护者有权对页面内容进行审核或调整结构。
- 如需删除或更新已有页面，请直接提交修改或联系维护者。
